# Cinema Kafka Streaming
**Лабораторная работа №1**  
*Тема: Потоковая обработка данных в реальном времени с использованием брокера Apache Kafka*

## Описание проекта
Разработан прототип системы для обработки событий в кинотеатре. В роли событий выступают операции продажи билетов на сеансы.

**Логика работы:**
1.  **Producer** генерирует поток JSON-сообщений, имитирующих покупку билета (выбор фильма, зала, ряда, места, цены).
2.  Сообщения отправляются в топик Kafka `cinema`.
3.  **Consumer** читает сообщения из топика и передает их в модуль валидации.
4.  **Валидатор** проверяет структуру данных и корректность значений (цена > 0, ряд/место > 0). В зависимости от результата, сообщение маркируется как `VALID` или `NOT VALID` и выводится в консоль.

## Стек технологий
*   **Apache Kafka** — брокер сообщений (режим KRaft).
*   **Python 3.x** — язык реализации продюсера и консьюмера.
*   **kafka-python** — клиентская библиотека для взаимодействия с Kafka.

## Структура проекта и назначение файлов
```
kafka-cinema-lab/
├── generator.py      # Генератор контента: формирует случайные данные о продаже билета.
├── producer.py       # Продюсер: сериализует данные от generator.py в JSON и отправляет в Kafka.
├── consumer.py       # Консьюмер: подписывается на топик, получает сообщения и вызывает валидатор.
├── validator.py      # Валидатор: содержит бизнес-логику проверки целостности и корректности данных.
├── requirements.txt  # Список зависимостей Python.
└── README.md         # Технический отчет.
```

## Формат сообщения (JSON)

Продюсер генерирует сообщения следующей структуры:

```json
{
  "cinema_event": "ticket_sale",
  "film": "Oppenheimer",
  "hall": 2,
  "seat_row": 5,
  "seat_number": 12,
  "session_time": "2024-05-20 19:30:00",
  "ticket_price": 550,
  "timestamp": "2024-05-20T17:30:01.123456"
}
```

## Логика валидации
Валидатор (`validator.py`) выполняет следующие проверки:
1.  **Проверка структуры:** Наличие всех обязательных полей (`cinema_event`, `film`, `hall`, `seat_row`, `seat_number`, `session_time`, `ticket_price`).
2.  **Проверка типов и значений:**
    *   Цена билета (`ticket_price`) должна быть строго больше 0.
    *   Номер ряда (`seat_row`) и номер места (`seat_number`) должны быть строго больше 0.
Если сообщение не проходит валидацию, оно помечается как `NOT VALID`. Это позволяет эмулировать появление ошибочных данных в потоке.

## Инструкция по запуску и тестированию
### Предварительные требования
*   Установленная Java 17+.
*   Установленный Python 3.8+.
*   Скачанный и распакованный дистрибутив Apache Kafka (версия 4.2.0 и выше).

### Этапы запуска
#### 1. Настройка окружения Python
```bash
python -m venv venv
.\venv\Scripts\activate

pip install -r requirements.txt
```

#### 2. Инициализация и запуск Kafka (режим KRaft)

Запуск Kafka через KRaft mode в cmd из директории с Kafka.

```bash
bin\windows\kafka-storage.bat random-uuid

bin\windows\kafka-storage.bat format --standalone -t <ваш_UUID> -c config\server.properties

bin\windows\kafka-server-start.bat config\server.properties
```
*После запуска Kafka готова принимать подключения на `localhost:9092`.*

#### 3. Запуск Consumer (Открыть новое окно терминала)
Consumer необходимо запустить до Producer, чтобы он сразу начал обрабатывать входящий поток с самого первого сообщения (`auto_offset_reset="earliest"`).

```bash
.\venv\Scripts\activate
python consumer.py
```
*Консоль перейдет в режим ожидания сообщений.*

#### 4. Запуск Producer (Открыть новое окно терминала)

```bash
.\venv\Scripts\activate
python producer.py
```
*Продюсер начнет генерировать и отправлять по одному сообщению в Kafka-топик `cinema` каждые 2 секунды.*
